审核上架流程如图所示

操作

1、查看：点击进入【查看 MCP 服务页面】


2、上架：只有审核通过且上架状态为“未上架”的可以点击上架，已上架的服务可以点击发起工单点击下架

3、发起工单：

  • 悬浮上去有俩功能，下架和编辑，下架逻辑上面已阐述
  • 下架：服务处于上架状态可以点击下架
    • 点击后二次弹框提示：是否确认下架，下架后可能会影响使用方调用服务
      • 若点击确定则将服务下架
    • 若该 MCP 服务已有一条待审核或者待提交审核的下架工单，不能再次提交下架工单，点击下架后toast 提示：该 MCP 服务已有一条流程中的下架工单
    • 下架后重新上架，只能上架审核通过的最新版本，如果出现以下这种情况也不会有问题
  • 编辑：点击后进入编辑页面
    • 无论编辑任何内容，保存后审核状态都会生成一个工单，审核状态变成“待提交”，可以点击“提交审核按钮”
    • 此列表中的“编辑”每次编辑时都是基于当前上架的最新版本编辑，而不是基于上次编辑的版本，也就是一个 MCP 服务可以发起多个编辑工单
      • 若需要基于上次编辑的内容继续编辑，需点击工单列表中的“编辑”

4、删除：正在上架中的服务不允许被删除，可被删除时，点击删除后二次弹框提示“删除后将不可恢复也无法查看对应的服务调用数据，是否删除”。

  • 若点击确定则将服务删除
