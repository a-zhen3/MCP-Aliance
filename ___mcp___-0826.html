鉴权逻辑

鉴权分两种类型：内置通用鉴权和用户独立鉴权

  • 内置通用鉴权：指的是服务提供方填入一个公用的鉴权参数，使用方开通后无需填入其他鉴权参数即可调用
  • 用户独立鉴权：服务提供方在服务上架时指定需要使用方填写的鉴权key是哪些，服务使用方开通服务后需要填入对应的值

选择“用户独立鉴权”后，默认展示一行需要填写的key

  • 下拉选框：单选，有header 和  query 两个选项，指的是该鉴权参数需要放到请求代码的哪个部分
  • 输入框：需要填入鉴权  key name
  • 点击+可以添加一行鉴权信息，最多可以填写5个鉴权参数（达到5个后不显示+ ）
  • 点击 X 可删除对应的鉴权参数，最后一个鉴权参数不可删除（不显示 X）
